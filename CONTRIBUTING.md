# Contributing to G2W

G2W is a relationship protocol. That means the way you contribute matters as much as what you contribute.

---

## The Vibe

G2W exists because AI development kept breaking down at the trust layer. Every contribution should make that trust stronger — cleaner commands, sharper docs, better hooks, tighter workflows. If a change adds complexity without adding trust, it doesn't belong here.

---

## How to Contribute

### Reporting Issues

Open a GitHub issue. Be specific:
- What command or feature is broken
- What you expected to happen
- What actually happened
- Your platform (Mac / Windows / Linux) and terminal

### Suggesting Features

Open an issue with the `[idea]` prefix in the title. Describe the problem it solves, not just the feature itself. G2W is about process — make the case for why this improves the relationship between user and AI.

### Pull Requests

1. Fork the repo and create a branch from `main`
2. Keep changes focused — one thing per PR
3. Update any docs that describe what you changed
4. PRs that add features without updating docs will not be merged

---

## The Trust Layer Applies Here Too

- Be honest about what your change does and why
- Don't scope-creep a PR — if you're fixing a bug, fix the bug
- If you're unsure about something, ask in the issue before building it

---

## Code of Conduct

Be direct. Be respectful. No bullshit.
